Paso 1 de 6
¿Qué es Hydra Framework?

Cuando construyes una startup, el 80% del trabajo inicial es siempre el mismo: login de usuarios, gestión de cuentas, pagos, permisos, registros de actividad… Hydra ya lo tiene construido y probado.

💡 Analogía: Piensa en Hydra como los cimientos y la estructura de un edificio. Tú no tienes que verter el hormigón ni levantar las paredes maestras — llegas y empiezas a decorar y amueblar tu piso directamente.

Lo que hace Hydra por ti: base de datos lista, autenticación lista, facturación lista, seguridad lista. Tu equipo empieza desde la capa de producto, no desde cero.

📋
Paso 2 de 6
¿Qué necesitas para empezar?

Casi nada. Hydra está pensado para que un desarrollador lo levante en menos de una hora.

🐳 Docker Desktop Descarga gratuita en docker.com — instala y listo. Sirve para ejecutar la base de datos sin configuración.
👨‍💻 Un desarrollador Cualquier dev backend o fullstack puede manejarlo. No requiere especialista en bases de datos.
📦 El repositorio Acceso al repo de GitHub: RubenGZ/Start-up-Pack. Tu dev lo clona en 30 segundos.
💳 Stripe (opcional) Solo si vas a cobrar desde el día 1. Cuenta gratuita en stripe.com, coges las claves y listo.
⚠️ Para founders no técnicos: muestra esta guía a tu CTO o dev de confianza. El tiempo estimado de setup es 30–60 minutos la primera vez.
🚀
Paso 3 de 6
Arrancar en 1 comando

Tu dev ejecuta esto y la base de datos arranca sola, aplica todas las migraciones y queda lista para conectar tu aplicación:

docker compose up -d
✅ Qué ocurre automáticamente
PostgreSQL 16 arranca en un contenedor aislado.
Se aplican las 24 migraciones del framework (usuarios, billing, auditoría…).
La base de datos queda lista en ~30 segundos.
💡 Analogía: Es como llegar a una oficina donde el electricista, el fontanero y el carpintero ya terminaron su trabajo. Tú solo enchufas los ordenadores y empiezas a trabajar.
🧩
Paso 4 de 6
Elige qué necesita tu startup

No tienes que usar todo. Hydra tiene 12 módulos — activas solo los que necesitas para tu idea. El resto no se instala.

  • 🔑
    authLogin por email mágico y Google. Sin contraseñas.
  • 👥
    usersPerfiles de usuario, equipos y roles (admin, miembro…).
  • 💳
    billingPlanes, suscripciones y portal de facturación con Stripe.
  • ❤️
    health-baseEstado del sistema en tiempo real para monitoreo.
  • 🛡️
    rate-limitProtección automática contra abuso de la API.
  • 📋
    auditHistorial de quién hizo qué y cuándo (RGPD-friendly).
  • 🎯
    onboardingFlujo de bienvenida multi-paso e invitaciones a equipos.
  • 🔔
    notificationsNotificaciones in-app, email y Slack automáticos.
  • 🗝️
    api-keysClaves de API para tus clientes, con revocación y uso.
  • 📊
    dashboardMétricas por organización: billing, actividad, API.
  • ⚙️
    baseFunciones comunes, UUIDs y timestamps. Siempre incluido.
  • 🌱
    seedDatos de prueba para demos y desarrollo. Solo en dev.
💡 ¿No sabes cuáles elegir? Para la mayoría de SaaS: auth + users + billing + health-base es el punto de partida ideal.

Paso 5 de 6
¿Qué tienes listo el Día 1?

Con los módulos básicos activados, tu startup ya dispone de todo esto sin escribir una sola línea de código:

  • ✅ Login seguro Magic links por email + OAuth Google, sin contraseñas que gestionar.
  • ✅ Multi-tenant Tus clientes tienen organizaciones separadas con roles (admin, miembro, viewer).
  • ✅ Facturación Planes free/pro/enterprise, upgrades, portal de cliente con Stripe.
  • ✅ Rate limiting Protección automática: cada plan tiene sus límites de uso definidos.
  • ✅ Auditoría completa Registro automático de cada acción para cumplimiento y debugging.
  • ✅ Health check Endpoint de estado del sistema para uptime monitoring desde el día 1.
  • ✅ Dashboard de métricas Datos de uso, facturación y actividad por organización en tiempo real.
  • ✅ Notificaciones Sistema de alertas in-app y email listo para conectar tu proveedor.
💡 En perspectiva: Normalmente este stack tarda entre 3 y 6 semanas en construirse desde cero. Con Hydra, tu equipo empieza a construir features de negocio desde el primer día.

Paso 6 de 6
Siguiente paso: conectar tu aplicación

Hydra es la base de datos y la lógica de negocio. Tu frontend o backend se conecta al SQL igual que a cualquier PostgreSQL estándar.

¿Qué hace tu dev a partir de aquí? Conecta tu framework favorito (Next.js, Django, Rails, Laravel…) a la base de datos usando la DATABASE_URL del .env. Todas las funciones de Hydra están disponibles como llamadas SQL normales.

🛠️ ¿Eres desarrollador?

Consulta la Guía técnica para desarrolladores → con todos los comandos, scripts y smoke tests paso a paso.

🔒 Convención de secretos: Las claves de Stripe, JWT y base de datos van en el .env local (nunca en el repositorio). En producción se configuran como variables de entorno en Railway, Render o Fly.io.
